Developer Journal Documentation

A comprehensive guide to using and making the most out of the Jover 15 Developer Journal

Features

What Our Website Offers

Create project

User can create several different project with own tag, contributors, and description.

Archive project

Project can be active or archived so that user can easily check which project is done or not.

Edit Project

User can edit project's TODO list and link to an issue for the project. TODO list allows Markdown grammar when edited

Create Log

User can create unique logs for each project. Logs include title, time, contributor, description, and code snippet using CodeMirror

View Calendar

User can see logs on a month, week, and day calendar. In day calendar, logs showcase title, editable progress bar, and code snippet.

Feedbacks

User can submit feedbacks through our unique sentiment widget

Usage

How to Use Our Website

Step 1

In main home_page, you can find the 5 buttons: "Home page", "Team page", "New", "Archive Page", and
"Give Feedback". Here, "New" button will create new project.
If you click "New" button, it will redirect to add_project_page.

Step 2

In this page, you can type project name, contributor, and description.
In addition, you have to select tags from 4 options. If you click "submit" button, project is created.
All of them should be filled up, otherwise it will give you alert message.

Step 3

Now, in the main home_page, you can see new project has been made.
If you click the project box, it will redirect to project_page.

Step 4

In project_page, you will see default text in TODO LIST and Link box.
You can edit by double-clicking each box or clicking the edit button. The red background
indicates that you are in edit mode. For TODO LIST, you can type either text or markdown:
Current available markdown is "#", "- [ ]", and "- [X]". The link doesn't have to include http:// or https://.
If you finish editing, then click the save button. The button will change to green confirm button for 3 secs.

Step 5

Each markdown text is shown in checkbox or bold text depending on what you wrote
In addition, https:// will be automatically appended to the link after you reload.
Now if you click "All Logs" button, it will redirect to month_page.

Step 6

In month_page, you can check the current day and other days in current month.
If you click each day in calendar, it will redirect to each day_page.

Step 7

In day_page, you can see time, title, description, progress bar, and code snippet
of logs for each day. You can simply add logs by clicking purple "+" button at the right top.

Step 8

In add_log_page, you can type title, time, contributor, description, and code snippet for the log.
All of them should be filled up, otherwise it will give you alert message.
After clicking submit button at the bottom, it will redirect back to day_page.

Step 9

Now you can see the updated log. You can also edit code_snippet by clicking the code_snippet box.
In addition, progress bar can be also adjusted, and it will be saved even though you leave the page.
Lastly you can also go to week_page and month_page by
clicking "week" and "month" button next to "+" button.

Step 10

In the week_page, you can check the title of logs for each day in current week.
Clicking each day section will redirect to day_page again.

Step 11

Going back to main home_page, if you click "archive" button inside the project box,
it will archive the page and remove it from the main home_page.
If you click "Archive Page" button at the top right, you can visit archive_page.
Even though it is archived, you can still access to the project_page and log pages for archived one.

Step 12

In main home_page, "Team Page" button will show our Jover 15's github page.
If this documentation website was useful, don't forget to revisit the developer journal website
and press "Give Feedback" button! Thank you.
